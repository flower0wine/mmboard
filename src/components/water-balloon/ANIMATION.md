## 概述

一个装满水的气球从空中下坠 → 触地变形 → 压扁 → 弹性回弹 → 静止，
循环播放。使用 **SVG path morphing**（路径形状渐变）+ **平移变换**实现。

| 属性 | 值 |
|------|----|
| 播放时长 | 5 秒 / 循环 |
| SVG ViewBox | `0 0 200 200`（宽 200px × 高 200px） |
| SVG 尺寸 | `width="200" height="200"` |
| 地面触点（底部着地位） | `y = 162`（全程统一） |
| 填充 | `#a8e6ff → #4aa8e0` 线性渐变 |
| 描边 | `#7fc9ed` 粗细 1px |

### SVG 坐标系统

SVG 的 y 轴**向下增长**（与数学坐标系相反）：
- y=0 为顶部，y=200 为底部
- 因此 `translateY = -70` 表示**向上**移动 70px
- 下坠时 translateY 从 -70 **增加到** -23 → 0，即从高往低落

## 代码结构：双层动画协同工作

动画由两个独立的 `<animate>` 元素在同一个 `<g>` 内并行驱动：

```xml
<g>
  <!-- ① 垂直平移：控制整体位置 -->
  <animateTransform
    attributeName="transform"
    type="translate"
    dur="5s"
    repeatCount="indefinite"
    values="0,-70; 0,-23; 0,0; 0,0"
    keyTimes="0; 0.25; 0.33; 1"
  />

  <!-- ② 路径变形：控制形状 -->
  <path fill="url(#wg)" stroke="#7fc9ed" strokeWidth="1">
    <animate
      attributeName="d"
      dur="5s"
      repeatCount="indefinite"
      values="形状A; 形状A; 形状B; 形状C; 形状D; 形状E"
      keyTimes="0; 0.25; 0.33; 0.42; 0.52; 1"
    />
  </path>
</g>
```

两个动画的 `dur` 相同（都是 5s），`repeatCount` 相同（无限循环），
因此它们始终保持时间同步。每一时刻的**实际视觉位置** =
**形状底部坐标** + **translateY 偏移量**。

### keyTimes 映射关系

```
时间  0%────25%────33%────42%────52%────────────100%
      │      │      │      │      │              │
tr:   -70    -23    0      0      0              0
      │下落→ │←变形→│←压扁→│←回弹→│←───稳定────→│
d:    滴A    滴A    静止   压扁   回弹           静止
                 ↑触地   ↑最扁   ↑过冲
```

两个动画的 keyTimes 数量不同，但始终并行：
- translate 在 0–33% 变化，之后保持 0
- path d 在 0–25% 保持水滴，25–33% 变形，之后继续变化

## 五次贝塞尔曲线（Cubic Bézier）快速入门

每条三次贝塞尔曲线由 4 个点定义：

```
P0 ──→ P1 ──→ P2 ──→ P3
起点   控制点1  控制点2  终点
```

- 曲线**从 P0 出发**，方向朝向 P1
- 曲线**在 P3 结束**，方向从 P2 来
- P1 和 P2 的**位置决定了曲线的弯曲程度**
- P1、P2 离 P0、P3 **越远**，曲线**越弯**

在本文的路径中，`C cx1,cy1 cx2,cy2 x,y` 表示：
- 起点 = 上一条命令的终点
- 控制点1 = `(cx1, cy1)`
- 控制点2 = `(cx2, cy2)`
- 终点 = `(x, y)`

---

## SVG 路径结构：为什么是 M-C-C-L-C-Z

所有 5 种形状必须使用**完全相同的命令序列**，才能让 SVG 对 `d` 属性做
线性插值（逐坐标点一一对应）。当前结构：

```
M  x1, y1              ← 移动到左下起点
C  x2,y2  x3,y3  x4,y4  ← ① 顶部与右侧弧线
C  x5,y5  x6,y6  x7,y7  ← ② 右下到底部
L  x8,y8                ← ③ 底部直线（零长度）
C  x9,y9 x10,y10 x11,y11 ← ④ 左下回到起点
Z                         ← 闭合
```

### 为什么 L 命令是零长度？

底部 L 命令的起止坐标相同（如 `L 100,185`），表示线段退化为一个点。
这是为了路径结构统一：下落时底部是圆弧（L 为零长度），落地后 L 展开为水平直线。

SVG 对 d 属性做线性插值时，L 的起止坐标各自与目标路径的 L 坐标对应，
因此 L 会从"一个点"平滑展开为"一条线"，或反之。

### 对称性

所有路径以 `x = 100`（viewBox 水平中点）为对称轴左右镜像。
右侧的坐标与左侧的坐标满足镜像关系：

```
右侧点 x_r  ↔  左侧点 x_l = 200 - x_r
```

## 动画五阶段

### 阶段 1: 下坠（0%–25%, 0–1.25s）

**垂直平移**: `translateY: -70 → -23`  
**形状**: 上窄下宽卵形，底部全圆弧

气球从 viewBox 上方进入（底部 `y = 185 + (-70) = 115`），
下落至地面（底部 `y = 185 + (-23) = 162`）。

```
M 45,135
C 65,20 135,20 155,135     ← 顶部弧线
C 158.6,155.7 158.45,185 100,185  ← 右侧到底部
L 100,185                    ← 零长度（底部一点）
C 41.55,185 41.4,155.7 45,135  ← 左侧到起点（镜像）
Z
```

| 关键点 | 坐标 | 含义 |
|--------|------|------|
| 顶部控制点 | (65,20) 和 (135,20) | 控制点越靠上（y 越小），顶部越窄 |
| 顶部峰值 | y ≈ 48.75 | 曲线最高点（三次贝塞尔中点公式） |
| 右侧衔接 | (158.6,155.7) | C1→C2 的公共控制点，保证切线一致 |
| 底部最宽 | 110px (x:45→155) | 水下沉导致底部更饱满 |
| 底部控制点 | (158.45,185) | C2 的末端控制点，控制底部弯曲 |
| 底部中点 | (100,185) | 零长度 L 的位置 |

---

### 阶段 2: 触地变形（25%–33%, 1.25–1.65s）

**垂直平移**: `translateY: -23 → 0`（同步补偿，见"触点统一"章节）  
**形状过渡**: 卵形 → 上圆下扁（静止水球）

变形期间的每一帧，形状底部从 185→162 下降 23px，
而 translateY 从 -23→0 上升 23px，两者恰好抵消，
地面触点始终在 y=162。

```
M 30,140
C 30,55 170,55 170,140     ← 顶部半圆拱
C 170,155 155,162 145,162  ← 右下圆角到平底
L 55,162                    ← 平底（不再是零长度）
C 45,162 30,155 30,140     ← 左下圆角
Z
```

| 关键点 | 坐标 | 含义 |
|--------|------|------|
| 顶部控制点 | (30,55) 和 (170,55) | 在端点正上方，形成垂直切线 |
| 顶端峰值 | y ≈ 76.25 | 比下落时低（水摊开了） |
| 底部 | y = 162 | 水平直线，表示接触地面 |
| 宽度 | 140px (x:30→170) | 比下落时宽 30px |

---

### 阶段 3: 压扁（33%–42%, 1.65–2.1s）

**垂直平移**: `translateY = 0`  
**形状**: 更宽更扁（冲击压缩到最大程度）

```
M 12,148
C 12,78 188,78 188,148
C 188,155 173,162 163,162
L 37,162
C 27,162 12,155 12,148
Z
```

| 对比 | 静止 | 压扁 |
|------|------|------|
| 宽度 | 140px | 176px (+26%) |
| 高度 | 85.75px | 66.5px (-22%) |
| 包围盒面积 | 14980 | ≈14875 (-0.7%) |

---

### 阶段 4: 回弹（42%–52%, 2.1–2.6s）

**垂直平移**: `translateY = 0`  
**形状**: 更高更窄（弹性恢复过冲）

```
M 40,130
C 40,37 160,37 160,130
C 160,150 148,162 142,162
L 58,162
C 52,162 40,150 40,130
Z
```

| 对比 | 静止 | 回弹 |
|------|------|------|
| 宽度 | 140px | 120px (-14%) |
| 高度 | 85.75px | 101.75px (+19%) |
| 包围盒面积 | 14980 | ≈14875 (-0.7%) |

---

### 阶段 5: 稳定（52%–100%, 2.6–5s）

形状恢复为静止（同阶段 2），停留 2.4 秒。
t=5s 时瞬间切回阶段 1 起点（水滴 + translateY=-70），开始新一轮。

> 为什么是"瞬间切回"？因为动画的 `repeatCount="indefinite"` 在
> dur 结束时立即从 t=0 重新开始，两个 `<animate>` 同时重置，
> 因此路径跳到水滴、平移跳到 -70，产生从天而降的效果。

## 核心技术原理

### 1. SVG Path Morphing 机制

```
<animate attributeName="d" values="路径A; 路径B; 路径C" keyTimes="0; 0.5; 1" />
```

SVG 在每两个相邻值之间做**线性插值**：将 `d` 字符串按空格/token 解析为
坐标数组，每个坐标值独立插值。例如：

```
路径A起点: M 45,135    路径B起点: M 30,140
  → 插值过程: x 从 45→30, y 从 135→140
```

**插值示例**（50% 进度）：

| 命令 | 路径A | 路径B | 插值结果(50%) |
|------|-------|-------|-------------|
| M | 45,135 | 30,140 | 37.5,137.5 |
| C | 65,20,135,20,155,135 | 30,55,170,55,170,140 | 47.5,37.5,152.5,37.5,162.5,137.5 |
| ... | ... | ... | ... |

这就是为什么所有路径必须有**完全相同的命令数量**（M + C + C + L + C + Z）。

### 2. G¹ 连续性（切线方向连续）

每段曲线衔接处，前一段的终点切线方向 = 后一段的起点切线方向。
没有方向突变，就不会出现"尖角"。

**右侧衔接点 (155,135) — C1 → C2：**

```
C1 终点切线: 控制点2(135,20) → 终点(155,135) = 向量 (20, 115)，角度 ≈ 80.1°
C2 起点切线: 起点(155,135) → 控制点1(158.6,155.7) = 向量 (3.6, 20.7)，角度 ≈ 80.1°
✓ 方向完全一致
```

**底部衔接点 (100,185) — C2 → C3：**

```
C2 终点切线: 控制点2(158.45,185) → 终点(100,185) = (-58.45, 0)，角度 180°（水平向左）
C3 起点切线: 起点(100,185) → 控制点1(41.55,185) = (-58.45, 0)，角度 180°（水平向左）
✓ 完全一致
```

**左侧衔接点 (45,135) — C3 → C1：**

与右侧对称，切线角度 ≈ -80.1°（对称镜像）。

### 3. G² 连续性（曲率连续）

一阶导数连续只保证切线方向一致，但曲率突变仍会产生视觉折感。
G² 要求二阶导数也成比例，让曲线弯曲程度也连续变化。

```
C1 二阶导数 (终点): B1''(1) = 6(P₃ - 2P₂ + P₁) = (-300, 690)
C2 二阶导数 (起点): B2''(0) = 6(Q₂ - 2Q₁ + Q₀) = (-58.5, 134.4)
比例: |(-300,690)| / |(-58.5,134.4)| ≈ 5.13  ✓ 方向一致
```

这意味着在衔接处：(a) 切线方向相同、(b) 弯曲方向相同、(c) 弯曲程度连续变化。

### 4. 切线速率比优化

即使方向一致，如果 C1 终点切线长度 |B'| = 116.7、C2 起点 |B'| = 21.0，
速率比 5.56:1 会在衔接处产生参数化速度突变。经过反复调试，此比值已从
原始的 8.3:1 降至 5.56:1，折感大幅减弱但未完全消除（因受 G² 约束无法继续提升）。

### 5. 面积守恒

各阶段包围盒（宽度 × 最大高度）保持约 14980：

| 阶段 | 宽 | 高(峰值→底部) | W×H | 相对静止 |
|------|----|--------------|------|---------|
| 下落 | 110px | 136.25px (48.75→185) | 14988 | +0.05% |
| 静止 | 140px | 85.75px (76.25→162) | 14980 | — |
| 压扁 | 176px | 66.5px (95.5→162) | ≈14875 | -0.7% |
| 回弹 | 120px | 101.75px (60.25→162) | ≈14875 | -0.7% |

> **注意**：W×H 是包围盒面积的近似值，并非形状精确面积。但比值可反映总体变化趋势。
> 包围盒面积相差 0.7% 以内，可认为近似守恒。

### 6. 地面触点统一

形状底部 y 坐标 + translateY 偏移量，在任何时刻都等于地面 y=162：

```
实际触点 = shape_bottom_y + translateY
```

| 时段 | shape 底部 y | translateY | 实际触点 | 说明 |
|------|-------------|-----------|---------|------|
| 下坠中 | 185 | -70 → -23 | 115 → 162 | 逐渐下降 |
| 下坠末 (t=0.25) | 185 | -23 | 162 ✅ | 刚好着地 |
| 触地变形 (0.25→0.33) | 185 → 162 | -23 → 0 | 162 ✅ | 同步抵消 |
| 压扁 / 回弹 / 稳定 | 162 | 0 | 162 ✅ | 贴地不动 |

translateY 从 -23 到 0 的补偿量与形状底部从 185 到 162 的下降量
完全相等（都是 23px），因此地面触点不跳变。

### 7. overflow-visible

SVG 默认超出 viewBox 的部分会裁剪。下坠初期气球顶部会超出 viewBox
（y<0），因此必须设置：

```tsx
<svg className="overflow-visible" ...>
```

## 线性渐变

```xml
<linearGradient id="wg" x1="0%" y1="0%" x2="100%" y2="100%">
  <stop offset="0%" stopColor="#a8e6ff" />  <!-- 浅蓝 -->
  <stop offset="100%" stopColor="#4aa8e0" /> <!-- 深蓝 -->
</linearGradient>
```

从左上到右下渐变，模拟水球的半透明质感。搭配 `stroke="#7fc9ed"` 描边
形成蓝色水球外观。

## 调试与调整建议

| 需求 | 修改位置 |
|------|---------|
| 调整下落高度 | `animateTransform` 首项 `0,-70` 中的 -70 |
| 加快/放慢 | 两处 `dur="5s"` 改为目标值 |
| 调整各阶段占比 | 同步修改两处的 `keyTimes` |
| 调整形状 | 修改 `values` 中的路径坐标（保持 M-C-C-L-C-Z 结构） |
| 调整弹性强度 | 修改压扁和回弹路径的宽/高 |
| 修改颜色 | 修改 `<linearGradient>` 的 `stopColor` 和 `stroke` |
| 改变底部高度 | 修改所有 5 个路径的底部 y 坐标 + translateY 补偿值 |
| 路径不自洽报错 | 检查所有路径的命令序列是否完全一致 |
